    Introduction

    About SAPAA

    The Stewards of Alberta's Protected Areas Association (SAPAA) applications help stewards, coordinators, and administrators monitor and track site inspections across Alberta's protected areas. Whether you're in the field or at your desk, SAPAA provides the tools you need to:

    • View all protected sites in one place
    • Track when sites were last inspected
    • Review inspection details and trends in naturalness scores
    • Monitor overall program health (Admins only)
    • Manage user accounts (Admins only)

    Who Should Use This Manual

    This manual is written for:

    • Stewards: Field workers who visit and inspect protected areas
    • Coordinators: Program managers who oversee multiple sites
    • Administrators: System managers who handle user accounts and overall program health

    You don't need to be technically experienced to use SAPAA. If you can browse websites and use a smartphone, you can use these applications.

    System Requirements

    Mobile App:

    • Android 5.0 or iOS 10.0 or higher
    • At least 100MB of free storage space
    • Internet connection (for initial download and syncing)
    • GPS capability (for map features)

    Web App:

    • Modern web browser (Chrome, Edge, Firefox, Safari)
    • Laptop or desktop computer recommended
    • Stable internet connection

    Understanding User Roles

    There are two types of users in SAPAA:

    Stewards (General Users):

    • View all protected sites
    • Read inspection details and history
    • Search and sort site information
    • Download sites for offline use (mobile app)
    • Generate site reports (mobile app)

    Administrators:

    • All steward features, plus:
    • Access admin dashboard with program-wide statistics
    • View data visualizations and heatmaps
    • Create, edit, and delete user accounts
    • Manage user roles and permissions

    How to Tell Your Role:

    • Mobile: Look for the "Admin Dashboard" option in the menu
    • Web: Look for the "Admin" button on the Protected Areas page

    If you don't see admin features, you have a steward account.

    Working Offline

    Purpose: Access site information when you're in remote areas without internet.

    Downloading Sites for Offline Use:

    1. While online, go to the Sites tab
    2. Tap the checkbox (‚òê) next to each site you want to download
      • You can select multiple sites
    3. Tap the "Download" button (appears when sites are selected)
    4. Wait for the download to complete
    5. A checkmark (‚úì) appears next to downloaded sites

    Using the App Offline:

    • When you lose internet connection, the app automatically switches to offline mode
    • Only downloaded sites will appear in your list
    • You can still:
      • View site details
      • Read inspection reports
      • Generate PDF reports

    Managing Storage Space:

    1. Go to the Sites tab while offline
    2. Select sites you no longer need offline
    3. Tap the "Delete" button
    4. Confirm deletion
    5. This frees up space on your device

    Tip: Download sites before heading to the field. Common strategies:

    • Download all sites in your assigned region
    • Download sites you plan to visit this month
    • Download frequently referenced sites

    Viewing Site Details (Mobile)

    Purpose: Get complete information about a specific protected area.

    Steps:

    1. From the Sites tab, tap on any site card
    2. The Site Detail screen opens

    Overview Section:

    • Site Name (large heading)
    • Type: E.g., Provincial Park, Natural Area
    • Region: Geographic area
    • Area: Size in hectares
    • Recreational Activities: Available activities at the site

    Naturalness Score:

    • Large number showing the average score (e.g., 3.4)
    • Color-coded slider:
      • Red (1.0): Poor condition
      • Yellow (2.0): Fair condition
      • Light Green (3.0): Good condition
      • Dark Green (4.0): Excellent condition
    • An indicator shows where this site's score falls on the scale

    Viewing Inspection Reports:

    There are two ways to view inspection data:

    Option 1: By Date

    1. Tap "By Date" at the top of the reports section
    2. You'll see a list of all inspections, newest first
    3. Each inspection shows:
      • Inspection date
      • Naturalness score for that visit
    4. Tap any inspection to expand it and see:
      • Steward name
      • Naturalness details (comments)
      • Individual observations (Q1, Q2, Q3, etc.)

    Option 2: By Question

    1. Tap "By Question"
    2. You'll see a list of all inspection questions (Q1, Q2, Q3, etc.)
    3. Each question shows:
      • Question ID and text
      • Number of responses across inspections
    4. Tap any question to see how answers have changed over time
    5. Responses are shown from newest to oldest

    When to Use Each View:

    • Use "By Date" when you want to read a complete inspection report
    • Use "By Question" when you want to track changes to a specific aspect over time (e.g., "How has trail condition changed?")

    Navigating Back:

    • Tap the back arrow (‚Üê) at the top left to return to the Sites list

    Using the Map

    Purpose: See where protected areas are located geographically.

    Steps:

    1. Tap the Map tab at the bottom
    2. The map loads showing all sites as pins
    3. Zoom in/out using pinch gestures
    4. Tap any pin to see the site name
    5. Tap "Open in Google Maps" to:
      • Launch the Google Maps app
      • Get navigation directions to the site

    Requirements:

    • Internet connection (to load map tiles)
    • GPS enabled (for showing your current location)

    Tip: Use this before a field visit to plan your route.

    Generating Reports

    Purpose: Create a PDF summary of a site's status and inspection history.

    Steps:

    1. Open any site's detail page
    2. Scroll to the bottom
    3. Tap the "Generate Report" button
    4. Wait a few seconds while the PDF is created
    5. The report opens in your device's PDF viewer
    6. From there, you can:
      • View the report on screen
      • Share it via email or messaging
      • Save it to your device

    What's Included in the Report:

    • Site overview (name, type, region, area)
    • Naturalness score
    • List of all inspections with dates and scores
    • Detailed observations from recent inspections

    Tip: Generate reports before going offline to have them available in the field.

    Comparing Inspection Data

    Purpose: Analyze inspection history in two different ways.

    Switching Views:

    Below the Naturalness Score section, you'll see two buttons:

    • View by Date
    • Compare by Question

    Click either to switch between viewing modes.

    View by Date:

    When to Use: Read complete inspection reports one at a time.

    What You'll See:

    • Heading: "Inspection Reports (X)" where X is the total count
    • Each inspection is a collapsible card showing:
      • Inspection date
      • Naturalness score for that visit (or "N/A")

    Steps:

    1. Click "View by Date"
    2. Click on any inspection card to expand it
    3. You'll see:
      • Steward Name: Who conducted the inspection
      • Naturalness Details: Free-text comments about site condition
      • Observations: Individual question responses (Q1, Q2, Q3, etc.)
    4. Each observation shows:
      • Question ID
      • Answer text from that inspection

    Tip: Use this to read through what a steward observed on a specific date.

    Compare by Question:

    When to Use: Track how a specific aspect has changed over time.

    What You'll See:

    • Heading: "Question Comparison (X questions)"
    • Each question is a card showing:
      • Question ID (e.g., Q3) in a badge
      • Question text/label
      • Number of responses (e.g., "3 responses across inspections")

    Steps:

    1. Click "Compare by Question"
    2. Click on any question card to expand it
    3. You'll see all responses to that question, sorted newest to oldest
    4. Each response shows:
      • Inspection date
      • Answer text

    Example Use Case:

    • Question: "Trail condition"
    • You can see: "Good" (2024), "Fair" (2023), "Good" (2022)
    • This shows the trail has been mostly well-maintained

    Tip: Use this to identify trends or recurring issues at a site.

    Using the Admin Dashboard

    Purpose: Monitor program-wide statistics and locate sites geographically.

    Who Can Access: Administrators only

    Steps to Access:

    1. From the Protected Areas page, click the "Admin" button (top right)
    2. You'll navigate to /admin/dashboard

    Dashboard Overview:

    The dashboard has four main sections:

    1. Header & Summary Stats:

    Two cards at the top show:

    • Total Records: Total inspection records in the database
    • Last Record: Date of the most recent inspection

    2. Naturalness Distribution (Pie Chart):

    • Shows how all inspections are distributed across naturalness categories
    • Each slice represents:
      • Excellent
      • Good
      • Fair
      • Poor
      • Cannot Answer
    • Slice colors match the condition (green for good, red for poor, etc.)

    Use This To:

    • See overall site health across the program
    • Identify if most sites are in good condition or need attention

    3. Top 5 Sites (Pie Chart):

    • Shows the five sites with the most inspection records
    • Each slice shows:
      • Site name
      • Total inspection count

    Use This To:

    • Identify heavily monitored sites
    • See which sites get the most attention from stewards

    4. Site Heatmap (Keyword Search + Map):

    Purpose: Locate sites geographically based on search terms.

    Steps:

    1. Type a keyword in the search box (e.g., "river", "park", part of a site name)
    2. Click "Search"
    3. The system:
      • Searches for matching sites in the database
      • Counts inspection records for each site
      • Uses geocoding (OpenCage API) to find coordinates
      • Plots sites on the map as a heatmap or markers

    What You'll See:

    • If sites are found: Map with colored points or heatmap
    • Below the search bar: "Found X locations for 'keyword'"
    • If no matches: "No sites found"
    • If geocoding fails: Message about locations not being available

    Tips:

    • Use short keywords (e.g., "lake" instead of "lakeside recreation area")
    • Try different variations if results are unexpected
    • Use this to identify geographic clusters of certain site types

    1. Log in and go to the Protected Areas page
    2. Click in the search box
    3. Type "Elk Island"
    4. The site appears in the filtered list
    5. Click the site card to view details

    Tip: You can search by partial name or by county/location.

    Scenario: Your supervisor asks when "Writing-on-Stone" was last visited.

    Mobile App:

    1. Search for "Writing-on-Stone" in the Sites tab

      • Inspection date is shown (e.g., "Sep 15, 2024")

    Web App:

    1. Search for "Writing-on-Stone" on the Protected Areas page
    2. Look at the site card:
      • Inspection date is shown
      • Or "No inspection date"
      • Last visit age (e.g., "2mo ago")

    Scenario: You want to see all past inspections for a site to identify trends.

    1. Open the site's detail page

    3. Choose your view:

    Web App:

    1. Open the site's detail page
    2. Scroll to the inspection section
    3. Toggle between:
      • View by Date: Read complete reports

    4. Click to expand and review details

    Tip: Use "By Question" to spot trends like increasing trail erosion or improving naturalness.

      • Shows count of sites not inspected for 2+ years
    3. Click "Sort" ‚Üí "Oldest First"

    5. Look for sites with:
      • üî¥ "Needs Review" badge

    Tip: Combine with search to find sites in your assigned region that need visits.

    Understanding Naturalness Scores

    What is a Naturalness Score?

    The Naturalness Score is an average value (typically 1.0 to 4.0) that summarizes how natural or disturbed a site is based on inspection reports.

    Score Ranges:

    • 1.0 ‚Äì Poor: Highly disturbed, significant human impact
    • 2.0 ‚Äì Fair: Some disturbance, but still functional
    • 3.0 ‚Äì Good: Mostly natural with minor disturbances
    • 4.0 ‚Äì Excellent: Pristine, natural condition

    How It's Calculated:

    • Each inspection includes a naturalness rating
    • The score shown is the average of all inspections
    • "N/A" means no scores have been recorded

    Why It Matters:

    • Helps identify sites that may need restoration or management
    • Tracks whether site condition is improving or declining over time
    • Guides resource allocation for conservation efforts

    Where to Find It:

    Mobile App:

    • On any site detail page, under the overview section
    • Color-coded slider shows where the score falls

    Web App:

    • On any site detail page, in the statistics cards
    • Gradient bar visualizes the score from red (poor) to green (excellent)

    Interpreting Trends:

    • If scores are declining: Site may be experiencing increased impact
    • If scores are improving: Management actions may be working
    • Use "Compare by Question" to see what specifically is changing

    Troubleshooting

    Mobile App Issues

    Problem: App Won't Open or Crashes on Launch

    Possible Causes:

    • Corrupted installation
    • Insufficient storage space
    • Outdated operating system

    Solutions:

    1. Restart your device
    2. Check for app updates:
      • Open the App Store or Play Store
      • Search for SAPAA
      • Tap "Update" if available
    3. Check storage space:
      • Go to device Settings ‚Üí Storage
      • Free up space if needed
    4. Reinstall the app:
      • Uninstall SAPAA
      • Restart your device
      • Reinstall from the store
      • Log in again

    Problem: Cannot Log In

    Possible Causes:

    • Incorrect email or password
    • Account not created yet
    • Network connection issue

    Solutions:

    1. Double-check your credentials:
      • Ensure Caps Lock is off
      • Verify email is spelled correctly
    2. Check your internet connection:
      • Try loading a website in your browser
      • Connect to Wi-Fi if available
    3. Contact your administrator:
      • They can verify your account exists
      • They can reset your password if needed

    Problem: Sites Not Loading or Showing "No Sites Found"

    Possible Causes:

    • No internet connection
    • No sites downloaded for offline use
    • Server issue

    Solutions:

    1. Check online/offline status:
      • Look at the badge at the top of the Sites tab
      • If offline, only downloaded sites appear
    2. Try syncing:
      • Go to the Analytics tab
      • Tap "Sync Now"
      • Wait for data to refresh
    3. Download sites for offline use (see "Working Offline" section)
    4. If problem persists: Contact your administrator

    Problem: Map Not Loading

    Possible Causes:

    • No internet connection
    • GPS disabled
    • Map service issue

    Solutions:

    1. Check internet connection: Map tiles require internet
    2. Enable GPS:
      • Go to device Settings ‚Üí Location
      • Turn on location services
      • Allow SAPAA to access location
    3. Try again later: Map services occasionally have outages

    Problem: Cannot Generate PDF Report

    Possible Causes:

    • Insufficient device permissions
    • Low storage space
    • Missing data

    Solutions:

    1. Check app permissions:
      • Go to device Settings ‚Üí Apps ‚Üí SAPAA
      • Ensure storage permissions are granted
    2. Free up storage space
    3. Ensure the site has inspection data: Reports require at least one inspection

    Web App Issues

    Problem: "Unable to Load Sites" Error

    Possible Causes:

    • Internet connection lost
    • Server maintenance
    • Database issue

    Solutions:

    1. Check your internet connection:
      • Try loading another website
      • Refresh the page
    2. Click "Try Again" on the error screen
    3. Clear browser cache:
      • Chrome: Settings ‚Üí Privacy ‚Üí Clear browsing data
      • Select "Cached images and files"
      • Click "Clear data"
    4. If problem persists: Contact your administrator

    Problem: "Unable to Load Site" on Detail Page

    Possible Causes:

    • Site was deleted
    • Database error
    • Incorrect URL

    Solutions:

    1. Click "Back to Sites"
    2. Search for the site again
    3. If it doesn't appear: The site may have been removed
    4. Contact your administrator if you believe the site should exist

    Problem: Cannot Log In

    Possible Causes:

    • Incorrect credentials
    • Browser cookies disabled
    • Account issue

    Solutions:

    1. Verify email and password:
      • Ensure Caps Lock is off
      • Check for typos
    2. Enable cookies:
      • Browser Settings ‚Üí Privacy
      • Allow cookies for the SAPAA site
    3. Try a different browser
    4. Contact your administrator for password reset

    Problem: Admin Dashboard Shows "No Data Available"

    Possible Causes:

    • No inspection data in database
    • Database permissions issue
    • Chart rendering error

    Solutions:

    • Refresh the page
    • Check other admin features:
    • If Account Management works, it's likely a data issue
    • Try the heatmap search:
    • Search for a known site
    • If it works, the database is accessible
    • Contact your administrator if charts remain empty

    Problem: Heatmap Returns "No Sites Found"

    Possible Causes:

    • Search term doesn't match any sites
    • Geocoding service issue
    • Database query error

    Solutions:

    • Try different keywords:
      • Use shorter terms (e.g., "lake" instead of "lakeside park")
      • Try partial names
    • Check spelling
    • Try a site you know exists
    • If the map appears but is empty: geocoding may have failed for those sites

    Problem: I Don't See the "Admin" Button

    Cause: Your account has the steward role, not admin.

    Solution:

    • Ask an existing admin to check your role
    • Go to Account Management (if you have another admin account)
    • Edit your user and change role to Admin

    Data Sync Problems

    Problem: Mobile App Shows Old Data

    Cause: Data hasn't synced recently.

    Solution:

    1. Go to the Analytics tab
    2. Tap "Sync Now"
    3. Wait for the sync to complete
    4. Check the "Last Inspection" date to confirm it updated

    Problem: Changes in Web App Don't Appear in Mobile App

    Cause: Mobile app data is cached and needs to be refreshed.

    Solution:

    1. Open the mobile app
    2. Go to Analytics ‚Üí "Sync Now"
    3. This pulls the latest data from the server

    Problem: Offline Data is Outdated

    Cause: Downloaded sites were cached before recent updates.

    Solution:

    1. Connect to the internet
    2. Go to the Sites tab
    3. Select the outdated sites
    4. Delete them from offline storage
    5. Download them again (this gets the latest data)

    Frequently Asked Questions

    Q1: What's the difference between a steward and an admin?

    A:

    • Stewards can view all sites, read inspection details, search and sort data, and generate reports (mobile app).
    • Admins have all steward features plus access to the Admin Dashboard (statistics, heatmap) and Account Management (create/edit/delete users).

    Q2: What does "Needs Attention" mean?

    A: A site is flagged "Needs Attention" (or "Needs Review") when it hasn't been inspected for approximately 2 years or more. This signals to coordinators that the site may need a new visit.

    Q3: Why do some sites say "No inspection date"?

    A: This means no inspection date has been recorded in the database for that site. Possible reasons:

    • It's a newly added site
    • It's an older record without a date
    • Inspections were conducted but dates weren't entered

    Q4: What is the Naturalness Score?

    A: The Naturalness Score is an average value (1.0 to 4.0) summarizing how natural or disturbed a site is based on inspection reports:

    • 4.0 = Excellent: Pristine, natural condition
    • 3.0 = Good: Mostly natural, minor disturbances
    • 2.0 = Fair: Some disturbance
    • 1.0 = Poor: Highly disturbed

    Higher values indicate better natural conditions.

    Q5: Can I change my own password?

    A: This depends on your deployment configuration. If you don't see a "Change Password" or "Reset Password" option in the app or web interface, contact your SAPAA administrator for assistance.

    Q6: How do I download sites for offline use?

    A: (Mobile app only)

    1. Go to the Sites tab while online
    2. Tap the checkbox next to sites you want to download
    3. Tap "Download"
    4. Sites are now available offline

    Q7: Why doesn't a site show up on the heatmap?

    A: Possible reasons:

    • The site name didn't match your search keyword
    • The site couldn't be geocoded (no coordinates found)
    • Temporary issue with the geocoding service

    Try a different keyword or contact an admin.

    Q8: How often should I sync the mobile app?

    A: It depends on your workflow:

    • Daily field use: Sync daily or before heading out
    • Occasional use: Sync weekly or before important meetings
    • After major updates: Sync immediately if you know new data was added

    Q9: Can I use the mobile app without internet?

    A: Yes, but with limitations:

    • You must download sites while online first
    • Only downloaded sites are available offline
    • You can view details, read inspections, and generate reports
    • You cannot sync new data or access the map

    Q10: Who do I contact if I have a problem?

    A: Contact your SAPAA program coordinator or system administrator using the contact information provided by your organization.

    Q11: How do I know which version of the app I'm using?

    A: (Mobile app)

    • Go to device Settings ‚Üí Apps ‚Üí SAPAA
    • The version number is listed there

    Q12: Can I access SAPAA from a tablet?

    A:

    • Mobile app: Yes, if you have an Android or iOS tablet
    • Web app: Yes, any device with a modern web browser can access it

    Q13: What happens if I lose internet connection while using the web app?

    A: You'll see error messages when trying to load data. Reconnect to continue. The mobile app is better suited for offline field work.

    Q14: How do I report a bug or suggest a feature?

    A: Contact your SAPAA coordinator with details about:

    • What you were trying to do
    • What happened instead
    • Which device/browser you were using
